# 営業管理アプリ セットアップガイド

このフォルダには、あなたの「営業管理.xlsx」を元にした、iOS・Android・Windows すべてのブラウザで動く
Webアプリ（PWA）が入っています。ホーム画面に追加すればアプリのように使えます。

無料で使える Google の Firebase というサービスをデータの保存先として使います。
以下の手順を順番に行えば、**約15〜20分**で複数人でリアルタイム共有できる状態になります。

---

## 1. Firebaseプロジェクトを作成する（無料）

1. ブラウザで https://console.firebase.google.com/ を開き、Googleアカウントでログインします。
2. 「プロジェクトを作成」→ 好きなプロジェクト名（例：`eigyo-kanri`）を入力 → 作成します。
   （Google アナリティクスは不要なのでオフのままで構いません）

## 2. Firestore データベースを有効化する

Firebaseコンソールの画面は時々デザインが変わります。2026年7月時点では、下記の名称・順番になっています。
表示が少し違っても、同じような意味の言葉を探してみてください。

1. Firebaseコンソールの**左側メニュー**から「**Databases & Storage**」（データベースと Storage）を探してクリックし、
   その中の「**Firestore**」を選びます。
   - 見つからない場合は、左メニュー上部の検索窓に「Firestore」と入力しても開けます。
2. 画面中央（または右上）にある「**データベースを作成**」または「**Add database**」というボタンをクリックします。
3. 「**Standard Edition**（Standard エディション）」が選ばれていることを確認して「**次へ**」をクリックします。
   （「Enterprise Edition」は選ばないでください）
4. 「**データベースID**」の入力欄が出ますが、何も変更せず、初期値（`(default)`）のまま「**次へ**」でOKです。
5. 「**ロケーション**」の一覧から `asia-northeast1`（東京）を選び、「**次へ**」をクリックします。
   - 一覧にない場合は `asia-northeast2`（大阪）など、日本に近いものを選んでください。
6. 「**セキュリティ ルール**」の開始モードを選ぶ画面が出ます。「**テストモード**」を選んで「**作成**」をクリックします。
   （この後の手順7で、正式なルールに上書きするので、ここでは「テストモード」のままで問題ありません）
7. データベースの作成が終わったら、画面上部のタブから「**ルール**」を開きます。
   今表示されている内容をすべて選択して削除し、代わりにこのフォルダの **`firestore.rules`** というファイルを
   テキストエディタで開いてその中身を全部コピーし、貼り付けます。最後に「**公開**」ボタンをクリックします。
   （これで「匿名ログインしたユーザーだけが読み書きできる」設定になります）

**つまずきやすいポイント**
- 「構築」というメニューは現在のFirebaseコンソールには無くなり、「Databases & Storage」に変わりました。
- 手順6で「本番環境モード」を選ぶと、手順7でルールを貼り替えるまでの間アプリが一切データを読み書きできずエラーになることがあるため、
  このガイドでは「テストモード」を選んだあとすぐにルールを上書きする手順にしています。

## 3. 匿名認証を有効化する

1. 左メニューの「構築」→「Authentication」を開く。
2. 「Sign-in method」タブ →「匿名」を選択して「有効にする」をオンにし保存。

## 4. アプリの接続情報（config）を取得する

1. 左上の ⚙️（プロジェクトの設定）→「全般」タブを開く。
2. 下の方の「マイアプリ」で「</> (ウェブ)」アイコンをクリックしてウェブアプリを登録
   （アプリ名は何でも構いません。Firebase Hosting の設定は今はチェック不要）。
3. 表示された `firebaseConfig = { apiKey: ..., authDomain: ..., ... }` の中身をコピーします。
4. このフォルダの **`firebase-config.js`** をテキストエディタで開き、
   `YOUR_API_KEY` などのプレースホルダーを、コピーした値に書き換えて保存します。

## 5. アプリを公開する（Firebase Hosting・無料）

パソコンに Node.js がインストールされていれば、以下のコマンドで公開できます。

```
npm install -g firebase-tools
firebase login
cd このフォルダのパス
firebase init hosting
  → 既存プロジェクトを選択（手順1で作ったプロジェクト）
  → 公開ディレクトリは "." （このフォルダ自身）を指定
  → 単一ページアプリ？ → いいえ
  → 既存ファイルを上書きしますか？ → いいえ（index.html等は上書きしない）
firebase deploy
```

デプロイが終わると `https://あなたのプロジェクトID.web.app` のようなURLが発行されます。
これが社内で使うアプリのURLです。

**Node.jsを使いたくない場合**：GitHub Pages や Netlify の「フォルダをドラッグ＆ドロップで公開」機能でも、
このフォルダをそのままアップロードするだけで動きます（`index.html` があるフォルダを丸ごとアップロードしてください）。

## 6. スマホ・PCのホーム画面に追加する

- **iPhone/iPad（Safari）**：発行したURLを開く → 共有ボタン → 「ホーム画面に追加」
- **Android（Chrome）**：URLを開く → 右上メニュー → 「ホーム画面に追加」／自動で追加バナーが出ることもあります
- **Windows（Edge/Chrome）**：URLを開く → アドレスバー右側の「インストール」アイコンをクリック

追加すると、アイコンをタップ・クリックするだけでアプリのように起動します。

## 7. 使い方

- 初回起動時に「新谷 壮央／白木 寿樹／川﨑 人志」から自分の名前を選びます（右上の「切替」でいつでも変更可）。
- 下部のタブから「ダッシュボード」「予定表」「TODO」「日報」「確定/見込」「取引先」を切り替えます。
- 各画面右上の「＋」ボタンから追加、一覧の「編集」「削除」から変更できます。
- 複数人が同時に開いていても、入力内容は自動でリアルタイムに反映されます。

## 補足：目標値・前年実績の初期値について

ダッシュボードの目標（月次売上・粗利目標）と前年売上実績は、元のExcelの「売上粗利表」シートの数値を
初期値として自動セットしています（初回起動時にFirestoreの `settings/main` ドキュメントとして保存されます）。
金額を変更したい場合は、Firebaseコンソールの Firestore Database 画面から
`settings` → `main` ドキュメントを直接編集してください（アプリ画面からの編集機能は含まれていません）。

## 注意事項

- このアプリは「匿名ログインしていれば誰でも読み書き可能」というシンプルな権限設定です。
  発行したURLは社内関係者以外に共有しないでください。
- より厳格なログイン（社員ごとのパスワード等）が必要な場合は、Firebase Authenticationの
  メール/パスワード認証への切り替えをご相談ください。
- 元の `営業管理.xlsx` はそのまま残っており、今後も参照用として使えます（アプリとは自動連携しません）。
